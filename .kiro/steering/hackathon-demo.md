# Hackathon - Preparación de Demo

## Estructura de la demo (3-5 minutos típico)

1. **Problema** (30 seg): ¿Qué problema resolvemos? Una frase clara.
2. **Solución** (30 seg): ¿Qué construimos? Descripción en una oración.
3. **Demo en vivo** (2-3 min): Mostrar el flujo principal funcionando.
4. **Impacto** (30 seg): ¿Por qué importa? Métricas o beneficios concretos.
5. **Cierre** (15 seg): Próximos pasos o visión a futuro.

## Tips para la demo

- Tener datos precargados, no perder tiempo llenando formularios en vivo
- Preparar un plan B (video grabado) por si la demo en vivo falla
- Practicar al menos 2 veces antes de presentar
- Mostrar el "wow moment" lo antes posible
- No mostrar código a menos que sea el diferenciador técnico

## Checklist pre-demo

- [ ] App desplegada y accesible
- [ ] Datos de prueba cargados
- [ ] Flujo principal probado de inicio a fin
- [ ] Video backup grabado
- [ ] Presentador asignado y preparado
- [ ] Timer configurado para no pasarse del tiempo
